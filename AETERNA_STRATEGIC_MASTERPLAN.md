## 1. エグゼクティブ・サマリー（CEO向け要約）

AETERNA Holdingsは、「MacBook 1台で全収益化領域を制圧する不滅の帝国」の実現に向け、3つのコア事業部を並行稼働させるフェーズに突入しています。本マスタープランは、帝国憲法が定める「社長第一主義」「収益最大化」「データ至上主義」「完全自律・完全無料」の4大原則に基づき、各事業部の収益化を最速かつ最小のCEO介入で達成するための戦略ロードマップです。

現在の優先事業と目標は以下の通りです：
1. **第一事業部（Affiliate.auto）**: ITエンジニア転職アフィリエイトでの月間33.7万円達成（自動化率95%）
2. **第二事業部（SaaS.auto）**: 時間管理SaaS「TimeTracker Pro」の3ヶ月以内MVPリリースと初期収益化
3. **統制・守備部門（Guardian）**: 24時間365日の自動監視とエラー時の自律復旧ロジックの完全稼働

## 2. 第一事業部：Affiliate.auto 戦略ロードマップ

### 2.1. 市場分析と収益ポテンシャル
ITエンジニア転職アフィリエイト市場は、依然として高単価（1件あたり1.5万円〜4.5万円）を維持しており、月間33.7万円の目標達成には、月間8〜23件のCV（コンバージョン）が必要です [1] [2]。TechGo（45,000円）やレバテック系（8,000円〜25,000円）を主力案件とし、SEO最適化された自動生成記事によるロングテールキーワードからの流入を狙います。

### 2.2. 戦略と実行計画
- **フェーズ1（1〜2週目）**: 既存の `master_autopilot.py` と GitHub Actions を活用した完全自動記事生成・投稿ラインの安定稼働。Gemini/Groq APIのフォールバック機構の強化。
- **フェーズ2（3〜4週目）**: 生成記事のSEO品質向上（RAGを用いた過去の成功記事パターンの学習）と、内部リンク構造の自動最適化。
- **フェーズ3（2ヶ月目以降）**: 検索順位データの自動取得と、低順位記事の自動リライト（Auto-Refinement）機能の実装。

## 3. 第二事業部：SaaS.auto 戦略ロードマップ

### 3.1. 市場分析と収益ポテンシャル
日本の勤怠管理・時間管理SaaS市場は拡大を続けており、エンジニア・フリーランス向けのニッチな市場にも十分な収益機会が存在します [3] [4]。競合（Clockify等）は多機能化・エンタープライズ化する傾向にあるため、AETERNAは「エンジニアの個人利用・小規模チーム」に特化し、シンプルさとAIによる自動レポート機能を武器に差別化を図ります。

### 3.2. 製品戦略：TimeTracker Pro
- **コアバリュー**: GitHub/VSCode連携による自動時間記録と、AIによる週次稼働レポートの自動生成。
- **価格モデル**: フリーミアムモデル。基本機能無料、AIレポート・高度な連携機能を含むプロプランを月額2,000円に設定。

### 3.3. 実行計画
- **フェーズ1（1ヶ月目）**: Next.js + SupabaseによるMVP（最小限の機能：手動時間記録、シンプルなダッシュボード）の開発とVercelへのデプロイ。
- **フェーズ2（2ヶ月目）**: Stripeによる決済基盤の統合。アーリーアダプター（Affiliate.autoの読者層から誘導）100名の獲得。
- **フェーズ3（3ヶ月目）**: AIレポート機能の実装と有料プランの開始。月間MRR 10万円の達成。

## 4. 統制・守備部門：Guardian システム強化計画

### 4.1. 目的と現状の課題
Guardianシステムは、帝国の全事業を24時間365日監視し、ダウンタイムを最小化する要です。現状は `health_check.py` による基本的な死活監視が実装されていますが、エラー時の自律復旧（Self-Healing）能力が不足しています。

### 4.2. 強化計画
- **監視対象の拡大**: サイトの死活だけでなく、APIのレートリミット、GitHub Actionsの実行エラー、Supabaseの接続状態を監視対象に追加。
- **自律復旧ロジック（Self-Healing）**:
  - API制限検知時：自動的に代替API（Gemini → Groq 等）に切り替える `api_fallback_manager.py` の統合。
  - デプロイ失敗時：直前の安定バージョンへの自動ロールバック機構の実装。
- **通知の最適化**: 軽微なエラーは自動修復してログに記録するのみとし、CEOへの通知は「自動修復不可能な致命的エラー」のみに限定（社長第一主義の徹底）。
