## 1. システムアーキテクチャ

### 1.1 全体フロー

```
毎日20時に自動実行
    ↓
[1] 市場分析エージェント
    ↓ (Gumroad売上ランキング、トレンド分析)
[2] コンテンツ企画エージェント
    ↓ (売れるテーマを決定)
[3] コンテンツ制作エージェント
    ↓ (電子書籍、テンプレート自動生成)
[4] 価格設定・販売エージェント
    ↓ (Gumroadにアップロード)
[5] 販売監視・改善エージェント
    ↓ (売上分析、改善提案)
[6] AIエージェント会議システム
    ↓ (複数エージェント間の協議)
[7] CEO向けダッシュボード更新
    ↓ (週次レポート自動生成)
```

### 1.2 データフロー

## 2. 各エージェントの詳細仕様

### 2.1 市場分析エージェント (`market_analyzer.py`)

**役割**: Gumroad内の売上トレンド、ユーザー需要を分析し、次に制作すべきコンテンツのテーマを提案

**入力**:
- Gumroad API（売上ランキング、ユーザーレビュー）
- 過去の売上データ
- 競合コンテンツ分析

**出力** (`market_analysis.json`):
```json
{
  "trending_categories": [
    {
      "category": "プログラミング",
      "trend_score": 95,
      "user_demand": "high",
      "avg_price": 2500,
      "estimated_monthly_revenue": 150000
    }
  ],
  "content_ideas": [
    {
      "theme": "Python初心者向けガイド",
      "target_audience": "未経験エンジニア",
      "estimated_demand": "high",
      "recommended_price": 1500,
      "priority": 1
    }
  ],
  "analyzed_at": "2026-05-11T20:00:00Z"
}
```

**実装要件**:
- Gumroad内の売上ランキングを毎日取得
- ユーザーレビューから需要を分析
- 過去の成功コンテンツのパターンを学習
- 推奨価格帯を自動計算

### 2.2 コンテンツ企画エージェント (`content_planner.py`)

**役割**: 市場分析結果に基づき、売れるコンテンツの企画を決定

**入力**:
- 市場分析結果（`market_analysis.json`）
- 過去の企画データ

**出力** (`content_plan.json`):
```json
{
  "planned_content": [
    {
      "id": "content_001",
      "theme": "Python初心者向けガイド",
      "description": "Pythonの基礎から実践まで、初心者向けの完全ガイド",
      "target_audience": "未経験エンジニア",
      "content_type": "ebook",
      "estimated_pages": 100,
      "recommended_price": 1500,
      "estimated_sales": 100,
      "estimated_revenue": 150000,
      "priority": 1,
      "deadline": "2026-05-12"
    }
  ],
  "planned_at": "2026-05-11T20:15:00Z"
}
```

**実装要件**:
- 市場分析結果を基に、3〜5個のコンテンツを企画
- 各コンテンツの売上予測を算出
- 優先度を自動決定
- 制作期間を見積もり

### 2.3 コンテンツ制作エージェント (`content_creator.py`)

**役割**: 企画に基づき、高品質な電子書籍、テンプレート、ガイドを自動生成

**入力**:
- コンテンツ企画（`content_plan.json`）

**出力**:
- Markdown形式の電子書籍
- Excel/Google Sheets形式のテンプレート
- PDF形式のガイド

**実装要件**:
- LLM（Gemini/Groq）を使用して、高品質なコンテンツを生成
- 構成: 目次 → 導入 → 本編（複数章） → まとめ → 付録
- 実用性を重視。コード例、テンプレート、チェックリストを含める
- 画像・図解が必要な場合は、テキスト形式で指示を記載

### 2.4 価格設定・販売エージェント (`pricing_and_sales.py`)

**役割**: コンテンツに最適な価格を決定し、Gumroadにアップロード

**入力**:
- 制作されたコンテンツ
- コンテンツ企画（価格推奨値）

**出力**:
- Gumroad上の公開ページ
- 販売開始ログ

**実装要件**:
- Gumroad APIを使用してアップロード
- 説明文、タグ、サムネイル画像を自動生成
- 価格は企画時の推奨値を採用
- 公開後、販売監視エージェントに通知

### 2.5 販売監視・改善エージェント (`sales_monitor.py`)

**役割**: 売上データを監視し、改善提案を生成

**入力**:
- Gumroad売上データ（API経由）
- ユーザーレビュー

**出力** (`sales_report.json`):
```json
{
  "daily_sales": [
    {
      "date": "2026-05-11",
      "content_id": "content_001",
      "sales": 15,
      "revenue": 22500,
      "reviews": 3,
      "avg_rating": 4.5
    }
  ],
  "improvement_suggestions": [
    {
      "content_id": "content_001",
      "issue": "説明文が不十分",
      "suggestion": "説明文を50文字追加し、具体的な利用シーンを記載",
      "expected_impact": "売上10%向上"
    }
  ],
  "monitored_at": "2026-05-11T23:00:00Z"
}
```

**実装要件**:
- 毎日23時に売上データを取得
- 売上が低いコンテンツを特定
- ユーザーレビューから改善点を抽出
- 改善提案を自動生成

## 3. AIエージェント会議システム

### 3.1 会議の流れ

毎日20時15分に「Gumroad事業部会議」を自動開催。以下の順序で進行：

1. **市場分析エージェント**: 本日のトレンド、売上ランキング変動を報告
2. **コンテンツ企画エージェント**: 明日以降に制作すべきコンテンツを提案
3. **コンテンツ制作エージェント**: 現在の制作状況、完成予定日を報告
4. **価格設定・販売エージェント**: 本日の販売状況、予定を報告
5. **販売監視・改善エージェント**: 前日の売上結果、改善提案を報告

### 3.2 会議での意思決定

各エージェントの提案を総合的に評価し、以下を自動決定：

```json
{
  "meeting_decision": {
    "date": "2026-05-11",
    "next_content_to_create": "content_002",
    "priority": 1,
    "deadline": "2026-05-12",
    "expected_revenue": 150000,
    "improvement_actions": [
      "content_001の説明文を改善",
      "content_001の価格を1500円から1800円に変更"
    ],
    "decided_at": "2026-05-11T20:30:00Z"
  }
}
```

### 3.3 会議システムの実装

**ファイル**: `meeting_system.py`

各エージェントの出力を読み込み、LLMを使用して会議を自動進行。最終的な意思決定結果を `meeting_decision.json` に保存。

## 7. 重要な注意事項

### 7.1 Gumroad API の利用

- Gumroad APIを使用して、売上データ、ユーザーレビュー、ランキングを取得
- API呼び出しは1日100回まで（無料枠）
- キャッシュを活用して、API呼び出し回数を最小化

### 7.2 LLM API の利用

- コンテンツ生成には Gemini API（メイン）、Groq API（フォールバック）を使用
- 無料枠の範囲内で運用
- APIレート制限に達した場合は、自動的に Groq に切り替え

### 7.3 データ保存とセキュリティ

- 全てのデータは `/home/ubuntu/AETERNA/gumroad.auto/system/data/` に JSON形式で保存
- 売上データ、ユーザー情報は暗号化して保存
- 毎日バックアップを実施
