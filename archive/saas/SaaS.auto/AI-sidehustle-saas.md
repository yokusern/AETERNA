# AI副業ツールSaaS「AutoSideHustle AI」プロジェクト設計書

> **目標**：初期構築後、週1〜2回のチェック＆改善だけで月50万円以上を安定的に稼ぐ recurring revenue システム

## 1. プロジェクト概要

- **商品名**：AutoSideHustle AI（副業自動化AI）
- **コンセプト**：自分自身が使っている「完全自動ブログ収益化システム」をユーザーに提供
- **価格プラン**：
  - ライト：980円/月（基本記事生成）
  - スタンダード（推奨）：2,980円/月（WordPress自動投稿＋レポート）
  - プレミアム：4,980円/月（優先処理＋専用サポート＋高度機能）
- **無料トライアル**：14日間（クレジットカード登録必須）

## 2. 収益化方法（自動化重視）

1. **メイン**：月額サブスクリプション（Stripe Billing）
2. **アップセル**：プレミアムプラン、追加ユーザー機能、専用テンプレート販売
3. **ブログ連動**：既存WordPressからSaaSランディングページへ自然誘導
   - CTA例：「この記事もAIで自動生成しました。同じ仕組みを自分のブログで回したい方はこちら」
4. **アフィリエイト**：ユーザー自身がブログで当SaaSを紹介できるアフィリエイトプログラム（将来的）

## 3. 技術構成

- **フロント**：Streamlit（MVP）→ 将来的に Next.js
- **バックエンド**：FastAPI
- **DB/認証**：Supabase（PostgreSQL + Auth + Storage）
- **ホスティング**：Render.com または Railway.app
- **AI**：Anthropic Claude（Sonnet 4.x メイン）＋必要時Gemini
- **スケジューラ**：GitHub Actions or Supabase Edge Functions
- **決済**：Stripe（サブスクリプション管理）
- **メール**：Resend or Supabase + Mailgun

## 4. コア機能（MVP）

- キーワード入力 → 高品質SEO記事生成（3000字以上、HTML出力）
- WordPress REST API自動投稿
- 最適Gumroad/Brainリンク自動挿入
- 週次パフォーマンスレポート（メール自動送信）
- ユーザーダッシュボード（生成履歴、利用状況）
- キーワードリサーチ簡易版

## 5. 構築手順（4週間計画）

### Phase 1（Week 1）：基盤構築
- Supabaseプロジェクト作成（Auth + Database）
- Stripeアカウント設定＋商品登録
- FastAPI + Supabase統合（ユーザー管理）

### Phase 2（Week 2）：AIコア機能
- 既存 `generate.py` をユーザー別実行にリファクタリング
- WordPress投稿機能の多ユーザー対応
- プロンプト最適化（RAGで過去成功記事参照）

### Phase 3（Week 3）：ダッシュボード・課金
- Streamlitダッシュボード
- Stripe webhookでサブスクリプション管理
- 無料トライアル自動処理

### Phase 4（Week 4）：統合・テスト・ローンチ
- 既存ブログに誘導リンク追加
- ランディングページ作成
- テストユーザー募集（X・note・ブログ）

## 6. 運用ルール（週1〜2回作業）

- **毎週月曜日**：生成記事サンプルチェック＋プロンプト微調整（30分）
- **毎週木曜日**：ユーザー数・解約理由確認＋新機能アイデア検討（1時間）
- **月1回**：新機能追加 or 大規模プロンプト更新
- エラー監視：GitHub Notifications + Slack

## 7. 集客戦略

1. 既存ブログ毎日自動投稿（継続）
2. 記事内CTA強化
3. X自動投稿（新記事＋SaaS告知）
4. ユーザー成功事例のブログ化（SEO強化）
5. 将来的：YouTube Shortsデモ動画

## 8. リスクと対策

- **AIコスト高騰** → キャッシュ活用＋Haiku併用＋ユーザー利用制限
- **Googleペナルティ** → ブログは集客用、メイン収益はSaaS
- **解約増加** → 週次価値提供を徹底
- **競合出現** → 「実際に月数十万稼いでいるシステム提供者」という実績で差別化

## 9. 成功指標（KPI）

- Month 3：有料ユーザー30人、MRR 8万円以上
- Month 6：有料ユーザー100人、MRR 25万円以上
- Churn Rate：15%以下
- ブログ検索流入：月間10,000セッション以上
